# auctionote 한 장 요약

## 프로젝트 한 줄

법원경매 WebSquare UI를 Playwright로 자동화해 물건 데이터를 수집하고, 후보 랭킹·휴리스틱 시세갭·지도 분포까지 보여주는 경매 분석 프로토타입.

## 왜 만들었나

- 부동산 경매 도메인 지식 학습
- 공공/준공공 사이트 크롤링 경험 축적
- 수집 데이터를 실제 판단 보조 정보로 변환하는 흐름 연습
- 미스고부동산 같은 경공매 데이터 제품과 맞닿는 포트폴리오 구축

## 핵심 구현

1. 수집

- `courtauction.go.kr` WebSquare UI를 Playwright headless Chromium으로 자동화
- 법원·용도·기간 조건 입력 후 렌더링된 결과 grid 파싱
- 14일 제한 대응용 sliding window 추가
- 서울 5개 법원 × 아파트/다세대주택/오피스텔/근린생활시설/토지 기본 타겟
- 보수적 pagination 지원

2. 데이터 모델링

- `AuctionItem` 스키마로 사건번호, 물건번호, 법원, 매각기일, 감정가, 최저가, 유찰횟수, 주소, 용도, 면적, 상태 정규화
- SQLite upsert 저장
- synthetic fixture 기반 parser 회귀 테스트 유지

3. 분석

- 유찰횟수별 평균/중앙 할인율
- 면적당 최저가
- 후보 랭킹 screening score
- 장기 유찰, 면적 누락, 유찰 대비 할인율 불일치 등 품질 플래그
- 지역·용도 그룹 내부 중앙값 기준 휴리스틱 시세갭

4. 시각화

- Streamlit 대시보드
- KPI, 차트, 후보 우선순위 테이블, 데이터 품질 체크
- 서울 구 중심점 기반 물건 위치 분포 지도

## 왜 미스고와 맞는가

미스고는 데이터맵, AI가치평가, 매물추천, 신건 알림처럼 경매 데이터를 "판단 가능한 정보"로 바꾸는 방향의 제품이다. `auctionote`도 단순 크롤러가 아니라 다음 흐름을 구현한다.

- 경매 물건 수집
- 데이터 정규화
- 가격/유찰/지역 기반 지표 계산
- 지도와 테이블로 탐색 가능하게 시각화

즉, "경매 데이터를 수집할 수 있나"뿐 아니라 "수집한 데이터를 어떻게 의사결정 정보로 바꾸는가"를 보여준다.

## 현재 강점

- WebSquare 기반 사이트를 실제로 자동화했다는 점
- fixture + pytest + mypy + ruff로 검증 가능한 구조라는 점
- 크롤링, 저장, 분석, 대시보드, 문서화까지 end-to-end로 닫혀 있다는 점

## 남은 한계

- DB에 커밋된 실데이터 스냅샷은 아직 37건으로 작음
- pagination은 라이브 재검증 필요
- 지도는 서울 구 중심점이라 정밀 좌표가 아님
- 휴리스틱 시세갭은 실거래가 기반이 아님
- 낙찰/기일 이력 미수집

## 다음 단계

1. 확장된 타겟으로 재수집
2. 실거래가 API 결합
3. 정밀 geocoding
4. 신건 알림/조건 저장

## 이력서용 요약 문장

법원경매 WebSquare UI를 Playwright로 자동화해 경매 물건 데이터를 수집하고, SQLite·Streamlit 기반 분석 대시보드를 구현했습니다. 유찰횟수, 면적당 가격, 휴리스틱 시세갭, 데이터 품질 플래그를 조합해 물건 검토 우선순위와 지도 기반 탐색 흐름을 설계했습니다.
