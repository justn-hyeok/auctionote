# 미스고 리서치 및 auctionote 보강 보고서

조사 기준일: 2026-04-24

## 1. 요약

미스고부동산은 전통적인 경매 정보지를 모바일·지도 중심으로 재구성한 경공매 프롭테크 서비스다. 핵심은 법원경매/온비드 물건을 단순히 보여주는 데서 멈추지 않고, 필지 지도, 실거래가, 등기·토지·건축물대장, 권리분석, 예상 수익률, 매물추천, 전문가 연결까지 묶어 사용자의 입찰 판단 비용을 낮추는 것이다.

auctionote는 현재 법원경매 WebSquare UI를 Playwright로 자동화해 실데이터를 수집하고, 유찰횟수별 할인율을 분석하는 작은 end-to-end 파이프라인이다. 미스고 지원용 포트폴리오로 강화하려면 수집 범위 확장보다 먼저 "수집한 경매 데이터를 판단 가능한 후보 정보로 바꾸는 능력"을 보여주는 편이 효과적이다.

이번 보강에서는 다음을 추가했다.

- 감정가 대비 할인율, 면적당 최저가, 지역·용도 내 상대 가격을 이용한 물건 검토 우선순위 점수
- 유찰 대비 할인율 불일치, 장기 유찰, 면적 누락 등 데이터 품질 플래그
- 서울 구 중심점 기반 오프라인 geocoding과 물건 위치 분포 지도
- 데이터셋 내부의 ㎡당 최저가 중앙값으로 계산한 휴리스틱 시세갭
- Streamlit 대시보드의 KPI, 검토 우선순위 테이블, 품질 체크, 지도 섹션
- 미스고 제품 방향과 auctionote 후속 로드맵을 연결한 본 보고서

## 3. auctionote 현재 상태

현재 구현은 다음 역량을 이미 보여준다.

- Playwright로 법원경매 WebSquare UI 자동화
- 법원·기간·물건분류 조건 입력 후 렌더링된 결과 grid 추출
- `AuctionItem` 스키마로 경매 물건 정규화
- SQLite 저장 및 upsert
- synthetic HTML fixture 기반 parser 회귀 테스트
- 유찰횟수별 평균/중앙 할인율 집계
- Streamlit + Plotly + Folium 대시보드
- pytest, ruff, mypy strict 검증

현재 데이터 스냅샷은 2026-04-22 수집 기준 서울 5개 법원, 2026-04-22부터 2026-05-06까지의 기일입찰 물건 37건이다. 용도는 아파트 35건, 오피스텔 1건, 다세대 1건으로 아직 편중되어 있다. 다만 기본 라이브 수집 타겟은 서울 5개 법원 × 아파트/다세대주택/오피스텔/근린생활시설/토지로 확장했고, 14일 제한을 넘는 기간은 sliding window로 나눌 수 있게 했다.

## 4. 이번 보강 내용

### 4.1 검토 우선순위 점수

`analysis/insights.py`를 추가해 각 물건을 0~100점의 screening score로 정렬한다. 점수는 투자 권유가 아니라 탐색 우선순위를 정하는 내부 분석 지표다.

사용한 신호:

- 감정가 대비 최저가 할인율
- 면적당 최저가
- 같은 지역·용도 내 면적당 최저가 중앙값 대비 비율
- 유찰 횟수 구간
- 주소/면적/source URL 완성도
- 매각기일 임박도

### 4.2 휴리스틱 시세갭

`analysis/market.py`를 추가해 현재 데이터셋 안에서 같은 지역·용도 그룹의 ㎡당 최저가 중앙값을 구하고, 이를 기준으로 물건별 추정 시세와 시세갭을 계산한다.

주의할 점은 이 값이 실제 감정평가나 실거래가 기반 시세가 아니라는 것이다. 외부 실거래가 API를 붙이기 전 단계에서, 데이터 제품의 랭킹·신뢰도·설명 구조를 검증하기 위한 휴리스틱 신호로 사용한다.

사용한 신호:

- 지역: 주소 앞 두 토큰 기준
- 용도: 아파트, 다세대, 오피스텔 등
- ㎡당 최저가 중앙값
- 표본 수 기반 신뢰도: low / medium / high
- `value_gap_rate = (추정시세 - 최저가) / 추정시세`

### 4.3 데이터 품질 플래그

경매 데이터는 원천 UI 변경, 파싱 오류, 특수 물건, 상태 변경 때문에 품질 검사가 중요하다. 다음 플래그를 추가했다.

- 감정가 누락
- 최저가 누락
- 최저가가 감정가 초과
- 면적 누락
- 유찰 대비 할인율 낮음
- 장기 유찰
- 진행/유찰 외 상태 확인 필요

특히 현재 데이터에는 유찰횟수 대비 할인율이 낮거나 장기 유찰인 물건이 있어, 단순 평균 차트보다 "왜 이 물건이 특이한가"를 설명하는 기능이 포트폴리오상 더 설득력 있다.

### 4.4 오프라인 geocoding

`analysis/geocode.py`를 추가해 서울 주소에서 구 이름을 추출하고, 서울 25개 구의 중심점으로 매핑한다. 네트워크 호출과 API 키가 필요 없는 대신 필지 단위 정밀도는 없다. 포트폴리오 데모에서는 안정적인 지도 시각화를 우선하고, 실제 제품화에서는 주소 정제와 필지 좌표 geocoding으로 교체하는 설계다.

### 4.5 대시보드 개편

Streamlit 대시보드에 다음 섹션을 추가했다.

- 물건 수, 평균 할인율, 중앙 ㎡당 최저가, 시세갭 15% 이상, 품질 확인 필요 건수
- 검토 우선순위 상위 15건과 시세갭/시세신뢰도
- 데이터 품질 체크 요약
- 서울 구 중심점 기반 물건 위치 분포

기존 유찰횟수별 할인율 차트와 물건 목록은 유지했다.

## 5. 미스고 맞춤 후속 로드맵

### 5.1 수집 안정화

우선순위가 가장 높다. 기본 타겟, sliding window, 보수적 pagination은 들어갔지만 라이브 재수집과 증분 기록이 남아 있다.

- 라이브 환경에서 페이지네이션 selector 재검증
- 확장된 용도 타겟으로 DB 재수집
- `crawl_runs`, `crawl_targets`, `raw_snapshots` 테이블 추가
- 검색 조건별 성공/실패, 수집 건수, fallback 여부 기록

이력서 포인트:

> WebSquare 기반 법원경매 사이트를 Playwright로 자동화하고, 법원·용도·기간 조합을 순회하는 증분 수집 파이프라인을 설계했습니다.

### 5.2 지도 기반 분석

미스고의 데이터맵 방향과 가장 직접적으로 연결된다.

- 현재는 서울 구 중심점 기반이므로 주소 geocoding을 실제 좌표로 교체
- 유찰횟수, 할인율, 용도, 품질 플래그별 지도 스타일링
- 구/동별 클러스터링
- 지도 선택과 테이블 필터 연동

이력서 포인트:

> 법원경매 물건 주소를 좌표화해 지도 기반 탐색 UI를 구현하고, 유찰횟수·할인율·용도별 공간 분포를 시각화했습니다.

### 5.3 실거래가·시세 결합

경매 판단의 핵심은 감정가 대비 할인율이 아니라 현재 시장가 대비 저평가 여부다.

- 국토교통부 실거래가 공개 데이터 또는 공공데이터포털 API 탐색
- 주소/법정동/면적/건물유형 기준 근접 거래 매칭
- 현재 휴리스틱 시세갭을 실거래가 기반 `최저가 / 추정시세`, `감정가 / 추정시세`, `시세 대비 안전마진`으로 교체
- 실거래 표본 부족 시 신뢰도 점수 제공

이력서 포인트:

> 경매 최저가를 주변 실거래가와 결합해 시세 대비 저평가 지표와 신뢰도 점수를 산출했습니다.

### 5.4 낙찰·기일 이력

미스고의 AI가치평가/고스톱류 기능과 가장 가까운 도메인 지식이다.

- 상세 페이지에서 회차별 기일 결과 수집
- 유찰/변경/취하/낙찰 상태 이력 저장
- 낙찰가율, 입찰자 수, 최종 낙찰까지 걸린 회차 분석
- 법원·용도·지역별 낙찰 경향 대시보드

이력서 포인트:

> 경매 기일 이력을 수집해 유찰 회차별 낙찰가율과 입찰 경쟁도를 분석했습니다.

### 5.5 검색·알림 UX

미스고의 대화형 검색, 맞춤형 신건 알림과 연결된다.

- 저장 조건: 지역, 용도, 최소 할인율, 최대 최저가, 유찰 횟수
- 새 수집 결과와 이전 DB diff
- 조건에 맞는 신건/가격변동 물건 알림
- 자연어 입력을 구조화된 필터로 변환하는 간단한 parser 또는 LLM wrapper

이력서 포인트:

> 관심 조건을 저장하고 새 경매 물건을 감지하는 신건 알림 로직을 구현했습니다.

## 6. 지원 서류용 문장

### 짧은 버전

법원경매 WebSquare UI를 Playwright로 자동화해 경매 물건 데이터를 수집하고, SQLite 저장소와 Streamlit 대시보드를 구현했습니다. 유찰횟수별 최저가율, 면적당 가격, 휴리스틱 시세갭, 데이터 품질 플래그, 후보 랭킹, 지도 분포를 제공해 경매 물건 탐색과 의사결정 보조 흐름을 end-to-end로 구성했습니다.

### 미스고 맞춤 버전
