# Live Crawl Notes

- **Target**: https://www.courtauction.go.kr (대한민국 법원경매정보)
- **Status (2026-04-22)**: **live crawl working** — `crawler/live.py` + `scripts/collect.py` populate `data/auctionote.db` from the real site.
- **Prior status (earlier on 2026-04-22)**: abandoned after a shallow probe. Root-cause of the initial bail was bad judgment, not a hard blocker. Details below.

## What actually works

1. Open `https://www.courtauction.go.kr/pgj/index.on` with Playwright (headless Chromium). WebSquare's JS bootstrap completes within ~6s and the full menu DOM is queryable.
2. Click `#mf_wfm_header_anc_auctnGdsMain` (물건상세검색) — this swaps the main frame to `PGJ151F00.xml` without changing the top-level URL.
3. Fill the search form: 법원, 기간(≤14 days — enforced by an `alert()`), 대분류 → 중분류 → 소분류. Example: 서울중앙 / 건물 / 주거용건물 / 아파트.
4. Click `#mf_wfm_mainFrame_btn_gdsDtlSrch`. After ~9s the grid `#mf_wfm_mainFrame_grd_gdsDtlSrchResult_body_table` fills.
5. Each auction record renders as **two `<tr>`s**: first row carries 법원+사건번호, 물건번호, 소재지+내역(면적 embedded), 비고, 감정가, 담당계/매각기일; second row carries 용도, 최저매각가, 진행상태(유찰 N회).
6. The 소재지 cell already embeds the 전용면적 as `[... 84.93㎡]`, so the list view is enough — no detail-page fetches needed.

Every search run is stamped with `WMONID`, `JSESSIONID`, `cortAuctnLgnMbr` cookies. No CAPTCHA was encountered during small-volume crawling (5 courts × one search each, 14-day window). Heavier loops should add rate-limiting between searches; current code inserts ~9s per search as a side effect of waiting for the grid to populate.

## Why the first attempt bailed

The initial Phase A probe was three `curl` calls:
- `GET /robots.txt` → 404, 29 KB HTML error page. No `Disallow`, but the response shape was treated as suspicious.
- `GET /` → 847 bytes, only a JS redirect to `/pgj/index.on`.
- `GET /pgj/index.on` → 2642 bytes of WebSquare shell. Empty `<body>`.

Conclusion drawn at the time: "WebSquare SPA → needs Playwright + session/XHR reverse → out of Phase A scope → pivot to synthetic fixtures." Two of those clauses were true; the third was a cop-out. The proper probe — actually open it in headless Chromium — takes about 30 seconds and would have shown the DOM rendering cleanly. Labeling the work "out of scope" without testing the cheapest validating tool was avoidance, not scoping.

Recorded here so the same shape of decision gets called out next time.

## Fallback behaviour

`scripts/collect.py` still falls back to parsing `fixtures/raw/detail_*.html` (the 6 synthetic fixtures generated by `scripts/seed_fixtures.py`) if `scrape_auctions` raises or returns zero records. The synthetic fixtures remain the regression test set for `crawler/parse.py` and are committed to the repo.

## Open items

- **Single-page results**: the grid only shows the first 10 records per search. For courts with more hits, need to click the pagination controls and merge pages. Currently accepts the 10-row cap silently.
- **2-week window cap**: enforced by the site. To cover more properties, iterate sliding windows. Not done yet.
- **용도 coverage**: default targets search only `아파트`. Add 단독주택 / 다세대주택 / 오피스텔 / 토지 / 근린생활시설 rotations to fill use_type variety.
- **Detail pages**: clicking the 소재지 link opens a detail page with 전용면적 explicitly, 감정평가서 link, 현황조사서, and per-기일 결과. Skipped for now; adopt if the list-cell regex for area turns out brittle.
- **Long-term**: the cleanest stable source is 공공데이터포털 **법원경매정보 Open API** (data.go.kr). When that becomes worth the API-key setup, swap `crawler/live.py` to HTTPX.
